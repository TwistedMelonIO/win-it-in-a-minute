# v3.5.0 Release Notes — Stream Deck OSC + Bulletproof License Persistence

**Released: April 29, 2026**

## What's New

### Stream Deck OSC Commands (via QLab)
- **Summary**: Three new OSC inputs that fire QLab cues to drive Stream Deck page changes after a Round 1 coin flip.
- **Details**: New addresses on the OSC listener (port 3001):
  - `/wiiam/sd/red` → fires QLab cue `SD2` (Red — Stream Deck page 2)
  - `/wiiam/sd/blue` → fires QLab cue `SD3` (Blue — Stream Deck page 3)
  - `/wiiam/sd/opposite` → fires whichever team's cue wasn't last used
- **Impact**: Coin-flip → server picks the right page once, then `/wiiam/sd/opposite` flips for the other team's turn. Cue identifiers are configurable via `SD_RED_CUE` / `SD_BLUE_CUE` env vars.

### Bulletproof License Persistence
- **Summary**: License key now survives every restart, rebuild, and even Docker volume destruction.
- **Details**: Two-tier persistence:
  - Primary: `/app/data/license_key` (Docker named volume)
  - Backup: `~/.twisted-melon/wiiam/license_key` (host bind mount, outside Docker entirely)
  - On every boot, server mirrors missing files between tiers — wiping the volume restores it from the host backup, and vice versa.
  - On every successful validation, the key is written to BOTH tiers.
  - Invalid keys never touch disk, so a bad paste in the web UI can't clobber a working license.
- **Impact**: Demo unit can no longer lose its license. Survives `docker compose down -v`, Docker Desktop factory reset, and machine reboots. Only deleting `~/.twisted-melon/` AND wiping the Docker volume in the same operation could lose it now.

## Bug Fixes
- Web UI activation no longer overwrites the persisted file before validation succeeds.
- Initial boot via `LICENSE_KEY` env var now persists the key to disk (previously it validated but never wrote, causing the key to be lost on next rebuild).

## Technical Changes
- New helpers in `server.js`: `syncPersistenceTiers()`, `persistLicenseKey()`, `safeRead()`, `safeWrite()`.
- `docker-compose.yml`: added `${HOME}/.twisted-melon/wiiam:/app/host-data` bind mount.
