# v3.3.0 Release Notes — Hardware-Based License Persistence

**Released: March 30, 2026**

## What's New

### Hardware-Based Machine ID
- **Summary**: License keys now survive full reinstalls on the same machine
- **Details**: The machine ID is now derived from the host Mac's hardware UUID (IOPlatformUUID) instead of the Docker container's virtual MAC address. This means the same physical machine always generates the same machine ID, regardless of whether the Docker volume, container, or entire project folder is deleted and recreated.
- **Impact**: Generate a license once per machine and it works forever — no need to request a new key after reinstalling

## Technical Changes
- `machine_id_simple.py`: Added `HOST_HARDWARE_ID` env var as priority 1 source for machine ID generation
- `docker-compose.yml`: Passes `HOST_HARDWARE_ID` from `.env` into the container
- `install_license.sh`: Detects host hardware UUID via `ioreg` (macOS) or `/etc/machine-id` (Linux) and writes it to `.env`
- Backwards compatible: existing installs with a persisted machine_id in the Docker volume continue to work unchanged

## Important Note
After upgrading to v3.3.0, run `install_license.sh` once to detect your hardware UUID. This will generate a **new machine ID** based on hardware — you'll need one new license key for this machine. After that, reinstalls will always produce the same machine ID.
